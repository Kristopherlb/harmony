## 1. Introduction

The Golden Observability Standard (GOS) defines the semantic dictionary and telemetry requirements for the ecosystem. By mandating a strict set of OpenTelemetry (OTel) attributes, the platform enables the Ops Agent to perform automated root-cause analysis, cost allocation, and business value tracking across the Runtime Tetrad.

## 2. Semantic Attribute Dictionary

All spans, logs, and metrics MUST include the following golden.* namespace attributes where applicable.

| Attribute Key | Description | Requirement |
| ------------- | ----------- | ----------- |
| golden.app_id | The unique identifier for the artifact/application. | Mandatory |
| golden.component_type | ORCHESTRATOR, REASONER, CONTRACT, or EXECUTABLE. | Mandatory |
| golden.business_value | Semantic impact (e.g., REVENUE_GENERATING, COMPLIANCE). | Optional |
| golden.data_classification | PUBLIC, INTERNAL, CONFIDENTIAL, RESTRICTED. | Mandatory |
| golden.cost_center | The budget code for the initiating user/team. | Mandatory |
| golden.initiator_id | The ID of the human or agent who started the process. | Mandatory |
| golden.is_compensation | Boolean indicating if the span is part of a Saga rollback. | Mandatory |

## 3. Log Format Specification

Logs MUST be emitted as structured JSON.

- **Required Fields:** timestamp, level, message, trace_id, span_id, golden.app_id.
- **Redaction Policy:** The Log Appender MUST automatically redact any field identified in the OCS SecretSchema.
- **Correlation:** Log lines MUST be correlated with the current Temporal RunID or Dagger ExecutionID via the trace_id.

## 4. Metrics Standard

The platform exposes metrics via Prometheus. All metrics MUST follow the naming convention: golden_{component}_{metric_name}.

- **Workflow Metrics:** golden_orchestrator_latency_seconds, golden_orchestrator_error_rate.
- **Agent Metrics:** golden_reasoner_tokens_consumed, golden_reasoner_iteration_count.
- **Capability Metrics:** golden_executable_compute_seconds, golden_executable_memory_peak_bytes.

## 5. Grafana JSON Model Baseline

Dashboards generated by the generate_golden_dashboard skill MUST conform to the following JSON structure:

- **Variables:** Must include $app_id, $environment, and $trace_id.
- **Panels:** Must prioritize the "Four Golden Signals" (Latency, Errors, Traffic, Saturation).
- **Annotations:** Workflow RETRY and COMPENSATION events MUST be overlaid on latency charts as annotations.
